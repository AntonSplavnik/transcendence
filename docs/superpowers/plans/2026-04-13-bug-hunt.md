# Bug Hunt: Ghost Crash in C++ Game Loop — Implementation Plan

> **Context for executing agents:** This plan covers three **independent** tracks for hunting a rare C++ game-core crash. Work tracks A and B are close to the same codebase and should land first; track C is long-running work that can proceed in parallel from the start. Do NOT wait for C before starting A/B.
>
> Each task block states its files, goal, and concrete edits. Tasks are coarser than typical TDD plans because this is debugging work — prioritize getting observability live over perfect test coverage. Commit often (after every numbered task).

**Goal:** Make the intermittent multi-player C++ game-core crash (a) observable, (b) non-fatal to the server, and (c) reproducible in under 10 minutes on a dev machine.

**Architecture:**
- **Track A (Observability):** Add a C++→Rust logging callback through the existing `cxx::bridge` in `backend/src/game/ffi.rs`. Sprinkle info/warn calls at key function boundaries in `game-core/src/`. Also write a thread-local "last log line" each call so a crash handler can dump it.
- **Track B (Crash containment):** Install a process-wide `SIGSEGV` + `SIGBUS` + `SIGABRT` handler using `libc::sigaction` with a thread-local `sigjmp_buf`. The `update_loop` calls `sigsetjmp` before each tick; on a non-zero return it logs the thread-local context and marks the game as crashed. Server lives.
- **Track C (Reproduction):** Build a tokio-based headless bot in the existing `bot/` crate. Bots register → login → create/join lobby → ready → pick class → send WebTransport input streams. One coordinator instance runs N bots, one designated master creates the lobby.

**Tech Stack:** Rust (tokio, reqwest, wtransport, cxx), C++ (CXX bridge, EnTT), Salvo backend.

**Bug profile (for debuggers reading this cold):**
- Only observed with ≥3 players actively inputting.
- 3/~10 repros in one session, then 0/~30 the next — strongly implies timing/race.
- No Rust panic, no server log → C++ segfault or abort taking the whole process down.
- Two repros happened minutes into a match, one happened within seconds of game start.
- Suspect areas in order of likelihood: (1) input handling for concurrent players, (2) attack chain / combat, (3) despawn/entity lifecycle, (4) event queue writes racing with `take_events`.

```

Then at the bottom of `ffi.rs` (after the `GameHandle` impl, not inside the `bridge` module) add the implementations and the thread-local context:

```rust
// =============================================================================
// C++ -> Rust logging callback
// =============================================================================

use std::cell::RefCell;

thread_local! {
    /// Last log line emitted from C++ on this thread. Read by the crash handler
    /// (Track B) via a raw pointer stored in TLS. Kept short (<=256 bytes) and
    /// updated on every call so it always reflects the most recent checkpoint.
    pub(crate) static LAST_GAME_LOG: RefCell<String> = const { RefCell::new(String::new()) };
    /// Lobby id (set once at thread start) for log correlation.
    pub(crate) static GAME_THREAD_LOBBY: RefCell<Option<String>> = const { RefCell::new(None) };
}

fn gc_log_info(msg: &str) {
    LAST_GAME_LOG.with(|slot| {
        let mut s = slot.borrow_mut();
        s.clear();
        s.push_str(msg);
    });
    GAME_THREAD_LOBBY.with(|slot| {
        let lobby = slot.borrow();
        match lobby.as_deref() {
            Some(id) => tracing::info!(target: "game_core", lobby = %id, "{msg}"),
            None => tracing::info!(target: "game_core", "{msg}"),
        }
    });
}

fn gc_log_warn(msg: &str) {
    LAST_GAME_LOG.with(|slot| {
        let mut s = slot.borrow_mut();
        s.clear();
        s.push_str(msg);
    });
    GAME_THREAD_LOBBY.with(|slot| {
        let lobby = slot.borrow();
        match lobby.as_deref() {
            Some(id) => tracing::warn!(target: "game_core", lobby = %id, "{msg}"),
            None => tracing::warn!(target: "game_core", "{msg}"),
        }
    });
}

/// Install the lobby id on the current game thread so subsequent log lines
/// from C++ are tagged. Call this once from the spawn closure in `manager.rs`
/// before entering `update_loop`.
pub fn set_game_thread_lobby(lobby_id: String) {
    GAME_THREAD_LOBBY.with(|slot| *slot.borrow_mut() = Some(lobby_id));
}
```

#include <cstdio>
#include <string>

namespace arena_game {

// Forward declarations of the Rust-exposed callbacks generated by CXX.
// These live in the cxx-generated header included from cxx_bridge.cpp.
void gc_log_info(::rust::Str msg);
void gc_log_warn(::rust::Str msg);

} // namespace arena_game

// Helper macros: format into a fixed stack buffer to avoid heap churn in hot paths.
// 256 bytes is enough for function-name + a few ids; truncates silently if over.
#define GC_LOG_INFO(fmt, ...) do { \
    char _gc_buf[256]; \
    int _gc_n = std::snprintf(_gc_buf, sizeof(_gc_buf), fmt, ##__VA_ARGS__); \
    if (_gc_n > 0) { \
        ::arena_game::gc_log_info(::rust::Str(_gc_buf, static_cast<size_t>(std::min<int>(_gc_n, sizeof(_gc_buf) - 1)))); \
    } \
} while (0)

#define GC_LOG_WARN(fmt, ...) do { \
    char _gc_buf[256]; \
    int _gc_n = std::snprintf(_gc_buf, sizeof(_gc_buf), fmt, ##__VA_ARGS__); \
    if (_gc_n > 0) { \
        ::arena_game::gc_log_warn(::rust::Str(_gc_buf, static_cast<size_t>(std::min<int>(_gc_n, sizeof(_gc_buf) - 1)))); \
    } \
} while (0)
```

# TRACK B — Catch Segfault in the Game Thread

**Why:** Once we can repro (Track C) we need to survive the crash cleanly so the log data from Track A actually gets flushed to stderr, and so the server stays up for additional repros in the same session.

**Approach:** Process-wide `sigaction` handler for `SIGSEGV`, `SIGBUS`, `SIGILL`, `SIGFPE`, `SIGABRT`. A thread-local `sigjmp_buf` is armed at the top of each tick; on signal, the handler writes a one-line crash banner + the `LAST_GAME_LOG` thread-local to stderr using `write(2)` (async-signal-safe), then `siglongjmp`s back to the tick checkpoint. The loop detects non-zero return and marks the game crashed.

**Caveats (read before coding):**
- `siglongjmp` out of a signal handler in C++ code with destructors-in-flight is undefined at the spec level. In practice on Linux/glibc it works and is the standard pattern for this kind of containment. We accept the caveat because the alternative (subprocess per game) is a multi-day refactor.
- After longjmp, the C++ game state is in unknown condition. We must **not** reuse the `GameHandle` — drop it, mark crashed, tear the lobby down.
- Install a `sigaltstack` so stack overflows can still be handled.

## Task B1 — Add the crash-handler module

**File:** `backend/src/game/crash_handler.rs` (new).

```rust
//! Per-thread crash containment for the C++ game loop.
//!
//! Installs a process-wide SIGSEGV/SIGBUS/SIGABRT/SIGILL/SIGFPE handler that,
//! for threads which have *armed* a sigjmp_buf via `arm_game_thread_checkpoint`,
//! dumps diagnostic info to stderr and siglongjmps back to that checkpoint.
//! Threads that have not armed a checkpoint fall through to the default
//! disposition (process aborts), preserving normal behavior everywhere else.

use std::cell::UnsafeCell;
use std::io::Write;
use std::sync::Once;

use libc::{
    SA_ONSTACK, SA_SIGINFO, SIGABRT, SIGBUS, SIGFPE, SIGILL, SIGSEGV, SS_DISABLE,
    c_int, c_void, sigaction, siginfo_t, stack_t,
};

// A thread-local flag+buffer. We cannot use `thread_local!` directly for the
// sigjmp_buf because we need a raw pointer to it inside an async-signal-safe
// handler. Use an UnsafeCell so we can take &mut through a raw pointer.

#[repr(C)]
pub struct ThreadCheckpoint {
    pub armed: bool,
    pub jmp_buf: libc::sigjmp_buf,
}

// SAFETY: only touched from the owning thread (armed path) and from the signal
// handler while that thread is stopped inside the handler — no concurrent access.
unsafe impl Sync for ThreadCheckpoint {}

thread_local! {
    static CHECKPOINT: UnsafeCell<ThreadCheckpoint> = UnsafeCell::new(ThreadCheckpoint {
        armed: false,
        jmp_buf: unsafe { std::mem::zeroed() },
    });
}

/// Call once at process startup (`main.rs`). Idempotent.
pub fn install() {
    static ONCE: Once = Once::new();
    ONCE.call_once(|| unsafe {
        install_altstack();
        for sig in [SIGSEGV, SIGBUS, SIGILL, SIGFPE, SIGABRT] {
            let mut sa: sigaction = std::mem::zeroed();
            sa.sa_sigaction = handler as usize;
            sa.sa_flags = SA_SIGINFO | SA_ONSTACK;
            libc::sigemptyset(&mut sa.sa_mask);
            libc::sigaction(sig, &sa, std::ptr::null_mut());
        }
    });
}

unsafe fn install_altstack() {
    const SIZE: usize = 64 * 1024;
    // Leak intentionally — lives for the process lifetime.
    let buf: &'static mut [u8] = Box::leak(vec![0u8; SIZE].into_boxed_slice());
    let mut ss: stack_t = std::mem::zeroed();
    ss.ss_sp = buf.as_mut_ptr() as *mut c_void;
    ss.ss_size = SIZE;
    ss.ss_flags = 0;
    libc::sigaltstack(&ss, std::ptr::null_mut());
    let _ = SS_DISABLE; // silence unused if libc changes
}

extern "C" fn handler(sig: c_int, _info: *mut siginfo_t, _uctx: *mut c_void) {
    // Async-signal-safe only. Do NOT allocate, do NOT take a lock, do NOT call tracing::.
    // Write a one-line banner and the thread-local "last log" via write(2).
    let sig_name: &[u8] = match sig {
        SIGSEGV => b"SIGSEGV",
        SIGBUS => b"SIGBUS",
        SIGILL => b"SIGILL",
        SIGFPE => b"SIGFPE",
        SIGABRT => b"SIGABRT",
        _ => b"SIG?",
    };

    let _ = write_stderr(b"\n!!! game-core crash caught: ");
    let _ = write_stderr(sig_name);
    let _ = write_stderr(b" last_log=");

    // SAFETY: LAST_GAME_LOG is a thread-local RefCell<String>. We're on the
    // same thread. The RefCell may already be borrowed, in which case we skip.
    // This is a best-effort dump.
    super::ffi::LAST_GAME_LOG.with(|slot| {
        if let Ok(s) = slot.try_borrow() {
            let _ = write_stderr(s.as_bytes());
        } else {
            let _ = write_stderr(b"<borrowed>");
        }
    });
    let _ = write_stderr(b"\n");

    // If this thread has armed a checkpoint, longjmp back to it.
    CHECKPOINT.with(|cell| unsafe {
        let cp = &mut *cell.get();
        if cp.armed {
            cp.armed = false;
            libc::siglongjmp(&mut cp.jmp_buf, sig);
        }
    });

    // Not armed: re-raise default disposition so the process still aborts,
    // preserving normal behavior for non-game threads.
    unsafe {
        libc::signal(sig, libc::SIG_DFL);
        libc::raise(sig);
    }
}

fn write_stderr(bytes: &[u8]) -> std::io::Result<()> {
    // SAFETY: fd 2 is stderr. `write` is async-signal-safe.
    unsafe {
        libc::write(2, bytes.as_ptr() as *const c_void, bytes.len());
    }
    Ok(())
}

/// Arm the per-thread checkpoint. Returns 0 on normal return, non-zero signal
/// number if unwound via siglongjmp.
///
/// Idiomatic use inside the game loop:
///
/// ```ignore
/// let rc = crash_handler::arm_game_thread_checkpoint();
/// if rc != 0 { /* crashed — log + break */ }
/// game.update();
/// crash_handler::disarm();
/// ```
#[must_use]
pub fn arm_game_thread_checkpoint() -> c_int {
    CHECKPOINT.with(|cell| unsafe {
        let cp = &mut *cell.get();
        cp.armed = true;
        libc::sigsetjmp(&mut cp.jmp_buf, 1)
    })
}

pub fn disarm() {
    CHECKPOINT.with(|cell| unsafe {
        let cp = &mut *cell.get();
        cp.armed = false;
    });
}
```

Declare the module: in `backend/src/game/mod.rs` add `pub mod crash_handler;`.

Make `LAST_GAME_LOG` in `ffi.rs` `pub(crate)` (already set in Task A1) so the handler can read it.

# TRACK C — Headless Bot for Reproduction

**Why:** We need to trigger the crash on demand. A swarm of 3-8 bots spamming inputs in one lobby is the smallest repro scaffolding that matches the bug profile (multi-player concurrent input).

**Design tenets:**
- One `bot` process → multiple bot identities → shared orchestrator.
- One bot is elected **master** at startup: creates the lobby, picks gamemode. Others join.
- Inputs are driven by a configurable action table: each action has a weight and a duration range; every tick each bot either continues its current action or rolls a new one.
- `reqwest::Client` for HTTP; `wtransport::Endpoint` for the gameplay stream. Serialization matches whatever `GameClientMessage` uses (check `messages.rs` for the serde layout — JSON or bincode).

### Live Server Verification Loop — READ BEFORE WRITING CODE

There is a **working production instance** of the backend running at `https://transcendence.kajus.cloud`. The LLM(s) implementing this track are expected to treat this server as the primary integration-test target: after each feature task (C1 → C6), rebuild and run the bot binary against that URL to verify the feature actually works end-to-end, then adjust based on real responses. Do **not** batch all tasks and test only at the end — the whole point is that each task's assumptions (JSON field shapes, header names, WebTransport handshake, etc.) are cheaply falsifiable against a live backend in under 10 seconds.

Three non-negotiable implications for how the bot must be built:

1. **The binary must always exit on its own within a bounded time window.** If the LLM runs `cargo run -p bot -- ...` and the program never exits, the tool harness never receives output and the LLM cannot see what happened. Every run must terminate. See the `--duration-secs` and `--stop-after` flags in Task C1 — the default duration is **30 seconds** for exactly this reason. 0 means "run until killed" and is opt-in only for the human-driven crash hunt in Task C9.

2. **The binary must produce an end-of-run summary via `tracing::info!`.** Each bot emits a structured `info` line with all milestone booleans, input count, and exit reason. The swarm emits one `info` line with `overall=PASS` or `overall=FAIL`. At default log level these always appear. A human doing long crash-hunt runs can set `RUST_LOG=bot=warn` to suppress them and only see problems. Any non-zero exit code means a milestone failed; the summary explains which one.

3. **`--accept-invalid-certs` must propagate to BOTH the HTTP client (reqwest) AND the WebTransport client (wtransport).** The flag defaults to `false` (live server has a real cert), but local dev builds use self-signed certs and need this flag for both transports.

**Live server hygiene:**
- The live server has a **valid TLS certificate**. Do not default to `--accept-invalid-certs`. That flag is local-dev-only.
- Bot accounts persist between runs (the live DB is persistent). Username `bot_0000` etc. will already exist after the first run — `Api::register` already handles the 409-conflict case as success, but re-verify this against the live server's actual response body.
- If a previous run crashed mid-match on the live server, the server cleanly ends the match on its own (it has Track B's crash handler in production already — that's why we can't reproduce the bug on the live server even though it happens there). The bot should still handle "already in a lobby" style errors idempotently.
- **Do not stress the live server.** Iterative verification runs use 1–4 bots and run for ≤30s each. Do not point a 30-bot swarm at `transcendence.kajus.cloud` just to see what happens.
- **Do not commit any bot credentials.** Use a hardcoded dev password like `"botpass123!"` — rotate if you ever point at a production-sensitive deployment.

### Log Level Conventions — FOLLOW EVERYWHERE

All bot output goes through `tracing`. No `println!`, no `eprintln!`, no `dbg!`.

| Level | What goes here | When a human sees it |
|---|---|---|
| **Error** | A phase failed and the bot cannot continue. Auth rejected (non-409), lobby create 500, swarm abort, bot run panic. | Always visible. One error = one broken thing to investigate. |
| **Warn** | Unexpected but survivable. Stream send failure (bot exits play but summary still emits), HTTP 409 on register (expected edge case), deadline grace exceeded. | Visible at `RUST_LOG=bot=warn` and above. Crash-hunt mode. |
| **Info** | Key lifecycle events — one line per significant state transition. `"bot registered"`, `"bot logged in"`, `"master created lobby"`, `"bot stream connected"`, `"bot play complete"`, per-bot summary line, `"swarm complete overall=PASS"`. | Visible at default `RUST_LOG=info`. Iterative verification mode — the LLM reads these. |
| **Debug** | Per-tick noise: input frames sent (every Nth tick), action state transitions, full HTTP request/response, WebTransport handshake details, barrier sync timing. | Only visible at `RUST_LOG=bot=debug`. Bot development / transport debugging. |

Default env filter in `main.rs`: `"info"`. Users can override with `RUST_LOG=bot=debug` for verbose, `RUST_LOG=bot=warn` for quiet crash-hunt runs.

```rust
use rand::Rng;
use serde::Serialize;

#[derive(Serialize, Debug, Clone, Default)]
#[serde(rename_all = "snake_case")]
pub struct InputFrame {
    pub movement: [f32; 3],
    pub look_direction: [f32; 3],
    pub attacking: bool,
    pub jumping: bool,
    pub ability1: bool,
    pub ability2: bool,
    pub dodging: bool,
    pub sprinting: bool,
}

/// A single scheduled action the bot is currently executing. When the
/// remaining tick count hits 0 the bot rolls a new action.
pub struct ActionState {
    current: Action,
    ticks_remaining: u32,
}

#[derive(Clone, Debug)]
pub enum Action {
    Move { dir: [f32; 3], ticks: u32 },
    Jump { ticks: u32 },
    Attack { ticks: u32 },
    Ability1 { ticks: u32 },
    Ability2 { ticks: u32 },
    Dodge { ticks: u32 },
    Idle { ticks: u32 },
}

impl ActionState {
    pub fn new_random<R: Rng>(rng: &mut R) -> Self {
        Self { current: Self::roll(rng), ticks_remaining: 0 }
            .rolled(rng)
    }

    fn rolled<R: Rng>(mut self, rng: &mut R) -> Self {
        self.current = Self::roll(rng);
        self.ticks_remaining = Self::ticks_for(&self.current);
        self
    }

    fn roll<R: Rng>(rng: &mut R) -> Action {
        // Weighted: movement is the most common, so multi-player movement races
        // are the most likely repro path.
        match rng.gen_range(0..100) {
            0..=40 => Action::Move {
                dir: {
                    let theta = rng.gen_range(0.0_f32..std::f32::consts::TAU);
                    [theta.cos(), 0.0, theta.sin()]
                },
                ticks: rng.gen_range(15..90),
            },
            41..=55 => Action::Attack { ticks: rng.gen_range(3..12) },
            56..=65 => Action::Jump { ticks: rng.gen_range(1..3) },
            66..=75 => Action::Ability1 { ticks: rng.gen_range(3..8) },
            76..=85 => Action::Ability2 { ticks: rng.gen_range(3..8) },
            86..=92 => Action::Dodge { ticks: rng.gen_range(2..4) },
            _      => Action::Idle { ticks: rng.gen_range(10..40) },
        }
    }

    fn ticks_for(a: &Action) -> u32 {
        match a {
            Action::Move { ticks, .. } | Action::Jump { ticks } | Action::Attack { ticks }
            | Action::Ability1 { ticks } | Action::Ability2 { ticks } | Action::Dodge { ticks }
            | Action::Idle { ticks } => *ticks,
        }
    }

    /// Advance one tick and produce the input frame to send.
    pub fn step<R: Rng>(&mut self, rng: &mut R) -> InputFrame {
        if self.ticks_remaining == 0 {
            *self = std::mem::replace(self, Self::new_random(rng)).rolled(rng);
        }
        self.ticks_remaining -= 1;

        let mut f = InputFrame::default();
        match &self.current {
            Action::Move { dir, .. } => f.movement = *dir,
            Action::Jump { .. } => f.jumping = true,
            Action::Attack { .. } => f.attacking = true,
            Action::Ability1 { .. } => f.ability1 = true,
            Action::Ability2 { .. } => f.ability2 = true,
            Action::Dodge { .. } => f.dodging = true,
            Action::Idle { .. } => {}
        }
        f
    }
}

```rust
use anyhow::Result;
use rand::SeedableRng;
use rand::rngs::SmallRng;
use tokio::time::Instant;

use crate::api::Api;
use crate::config::{Config, Phase};
use crate::input::ActionState;
use crate::transport::GameStream;

pub struct Bot {
    pub index: usize,
    pub username: String,
    pub api: Api,
    pub rng: SmallRng,
}

/// Machine-readable milestone tracker. The coordinator collects one of
/// these per bot and emits the aggregate via `tracing::info!`.
#[derive(Debug, Clone, Default)]
pub struct BotSummary {
    pub index: usize,
    pub registered: bool,
    pub logged_in: bool,
    pub lobby_ok: bool,
    pub ready_ok: bool,
    pub stream_ok: bool,
    pub inputs_sent: u64,
    pub exit_reason: String,
    pub target_phase: String,
}

impl BotSummary {
    pub fn reached(&self, target: Phase) -> bool {
        match target {
            Phase::Register => self.registered,
            Phase::Login => self.logged_in,
            Phase::Lobby => self.lobby_ok,
            Phase::Ready => self.ready_ok,
            Phase::Stream => self.stream_ok,
            Phase::Play => self.stream_ok && self.inputs_sent > 0,
        }
    }
}

impl Bot {
    /// Constructs a new bot and runs register + login. Stops at Phase::Login
    /// if that's all the caller asked for. The new bot is returned even if
    /// stop_after <= Login so the coordinator can collect the summary.
    pub async fn new(cfg: &Config, index: usize) -> Result<(Self, BotSummary)> {
        let username = format!("bot_{:04}", index);
        let mut summary = BotSummary {
            index,
            target_phase: format!("{:?}", cfg.stop_after),
            ..Default::default()
        };

        let mut api = Api::new(cfg.base_url.clone(), cfg.accept_invalid_certs)?;
        api.register(&username, "botpass123!").await?;
        summary.registered = true;
        tracing::info!(bot = index, %username, "registered");

        api.login(&username, "botpass123!").await?;
        summary.logged_in = true;
        tracing::info!(bot = index, %username, "logged in");

        let bot = Self {
            index,
            username,
            api,
            rng: SmallRng::seed_from_u64((index as u64).wrapping_mul(0x9E3779B97F4A7C15)),
        };
        Ok((bot, summary))
    }

    /// Walks the bot through lobby → ready → stream → input loop, stopping
    /// at whatever milestone `cfg.stop_after` names. `deadline` is the hard
    /// wall-clock ceiling computed by the coordinator from `cfg.run_duration`.
    pub async fn run(
        &mut self,
        lobby_id: &str,
        cfg: &Config,
        deadline: Option<Instant>,
        summary: &mut BotSummary,
    ) -> Result<()> {
        if cfg.stop_after < Phase::Lobby {
            summary.exit_reason = "phase_target_login".into();
            return Ok(());
        }

        // Order matters: you must be in the lobby before you can pick a class
        // or flip ready. The master (index 0) is already a member implicitly
        // because it created the lobby, so skip the join call for it.
        if self.index != 0 {
            self.api.join_lobby(lobby_id).await?;
        }
        summary.lobby_ok = true;
        tracing::info!(bot = self.index, %lobby_id, "lobby joined");
        if cfg.stop_after < Phase::Ready {
            summary.exit_reason = "phase_target_lobby".into();
            return Ok(());
        }

        self.api
            .set_character(lobby_id, if self.index % 2 == 0 { "knight" } else { "rogue" })
            .await?;
        self.api.set_ready(lobby_id).await?;
        summary.ready_ok = true;
        tracing::info!(bot = self.index, "ready");
        if cfg.stop_after < Phase::Stream {
            summary.exit_reason = "phase_target_ready".into();
            return Ok(());
        }

        let mut stream = GameStream::connect(
            &cfg.base_url,
            self.api.jwt().expect("login succeeded so jwt is set"),
            lobby_id,
            cfg.accept_invalid_certs,
        )
        .await?;
        summary.stream_ok = true;
        tracing::info!(bot = self.index, "stream connected");
        if cfg.stop_after < Phase::Play {
            summary.exit_reason = "phase_target_stream".into();
            return Ok(());
        }

        // Full play loop — raced against the optional deadline.
        let mut action = ActionState::new_random(&mut self.rng);
        let mut ticker = tokio::time::interval(cfg.tick_duration);
        ticker.set_missed_tick_behavior(tokio::time::MissedTickBehavior::Skip);

        let sleep_fut = async {
            match deadline {
                Some(d) => tokio::time::sleep_until(d).await,
                None => std::future::pending::<()>().await,
            }
        };
        tokio::pin!(sleep_fut);

        tracing::info!(bot = self.index, "play loop started");

        loop {
            tokio::select! {
                () = &mut sleep_fut => {
                    summary.exit_reason = "duration_reached".into();
                    tracing::info!(
                        bot = self.index,
                        inputs_sent = summary.inputs_sent,
                        "play complete (duration reached)"
                    );
                    return Ok(());
                }
                _ = ticker.tick() => {
                    let frame = action.step(&mut self.rng);
                    if let Err(e) = stream.send_input(&frame).await {
                        summary.exit_reason = format!("stream_send_error: {e}");
                        tracing::warn!(bot = self.index, error = %e, "stream send failed");
                        return Ok(());
                    }
                    summary.inputs_sent += 1;
                    if summary.inputs_sent % 600 == 0 {
                        tracing::debug!(
                            bot = self.index,
                            inputs_sent = summary.inputs_sent,
                            "tick checkpoint"
                        );
                    }
                }
            }
        }
    }
}

/// Aggregate result of a swarm run. The coordinator emits a structured
/// summary via `tracing::info!` before returning.
#[derive(Debug)]

    pub lobby_id: Option<String>,
    pub bots: Vec<BotSummary>,

impl SwarmReport {

    }

    pub fn log_summary(&self) {

                registered = b.registered,
                logged_in = b.logged_in,
                lobby_ok = b.lobby_ok,
                ready_ok = b.ready_ok,
                stream_ok = b.stream_ok,
                inputs_sent = b.inputs_sent,
                reached_target = b.reached(self.target_phase),
                exit = %b.exit_reason,
                "bot result"
            );
        }
        let overall = if self.all_milestones_reached() { "PASS" } else { "FAIL" };
        tracing::info!(
            bot_count = self.bots.len(),
            lobby_id = self.lobby_id.as_deref().unwrap_or("<none>"),
            target_phase = ?self.target_phase,
            %overall,
            "swarm complete"
        );
    }
}

pub async fn run_swarm(cfg: Config) -> Result<SwarmReport> {
    tracing::info!(
        base_url = %cfg.base_url,
        bots = cfg.bot_count,
        mode = %cfg.mode,
        duration = ?cfg.run_duration,
        stop_after = ?cfg.stop_after,
        "swarm starting"
    );
    let mut report = SwarmReport {
        lobby_id: None,
        bots: Vec::with_capacity(cfg.bot_count),
        target_phase: cfg.stop_after,
    };

    // Compute the hard deadline once so every bot races the same clock.
    let deadline = cfg.run_duration.map(|d| Instant::now() + d);

    // 1) Register + login every bot sequentially (avoid hammering auth).
    let mut bots: Vec<Bot> = Vec::with_capacity(cfg.bot_count);
    let mut summaries: Vec<BotSummary> = Vec::with_capacity(cfg.bot_count);
    for i in 0..cfg.bot_count {
        match Bot::new(&cfg, i).await {
            Ok((bot, summary)) => {
                bots.push(bot);
                summaries.push(summary);
            }
            Err(e) => {
                tracing::error!(bot = i, error = %e, "bot register/login failed");
                let summary = BotSummary {
                    index: i,
                    target_phase: format!("{:?}", cfg.stop_after),
                    exit_reason: format!("auth_error: {e}"),
                    ..Default::default()
                };
                summaries.push(summary);
                report.bots = summaries;
                report.log_summary();
                return Ok(report);
            }
        }
    }

    if cfg.stop_after <= Phase::Login {
        for s in &mut summaries {
            if s.exit_reason.is_empty() {
                s.exit_reason = format!("phase_target_{:?}", cfg.stop_after).to_lowercase();
            }
        }
        report.bots = summaries;
        report.log_summary();
        return Ok(report);
    }

    // 2) Master (index 0) creates the lobby.
    let lobby_id = bots[0].api.create_lobby(&cfg.mode).await?;
    tracing::info!(%lobby_id, "master created lobby");
    report.lobby_id = Some(lobby_id.clone());

    // 3) Drive each bot through its remaining phases concurrently. We store
    //    summaries behind a Mutex so tasks can update their own slot.
    let lobby_id = Arc::new(lobby_id);
    let barrier = Arc::new(Barrier::new(cfg.bot_count));
    let summaries = Arc::new(Mutex::new(summaries));
    let cfg_arc = Arc::new(cfg);

    let mut handles = Vec::new();
    for (slot, mut bot) in bots.into_iter().enumerate() {
        let lobby_id = Arc::clone(&lobby_id);
        let barrier = Arc::clone(&barrier);
        let cfg = Arc::clone(&cfg_arc);
        let summaries = Arc::clone(&summaries);

        handles.push(tokio::spawn(async move {
            barrier.wait().await; // rough "start inputs" sync
            let mut local = {
                let s = summaries.lock().await;
                s[slot].clone()
            };
            if let Err(e) = bot.run(&lobby_id, &cfg, deadline, &mut local).await {
                tracing::error!(bot = bot.index, error = %e, "bot run failed");
                local.exit_reason = format!("run_error: {e}");
            }
            let mut s = summaries.lock().await;
            s[slot] = local;
        }));
    }

    // Wall-clock cap at the swarm level: if the bots haven't all exited by
    // deadline + a short grace window, we abort the remaining tasks so main
    // still prints and returns.
    let swarm_fut = async {
        for h in handles {
            let _ = h.await;
        }
    };

    if let Some(d) = deadline {
        let grace = std::time::Duration::from_secs(3);
        let total = d - Instant::now() + grace;
        let _ = tokio::time::timeout(total, swarm_fut).await;
    } else {
        swarm_fut.await;
    }

    let final_summaries = Arc::try_unwrap(summaries)
        .map(|m| m.into_inner())
        .unwrap_or_else(|arc| {
            // Fallback if the mutex is still shared — should not happen
            // because all spawned tasks have exited via the timeout/join.
            futures_util::executor::block_on(async move { arc.lock().await.clone() })
        });
    report.bots = final_summaries;
    report.log_summary();
    Ok(report)
}
```

A few notes for the implementing LLM:
- `futures_util::executor::block_on` on the fallback path requires `futures-util` (already in the Cargo.toml from Task C1). If clippy or the workspace disallows it, the simpler and equally correct alternative is to ensure tasks all finish (they do, given the grace window) and unwrap unconditionally with `.expect("all tasks joined")`.
- `Phase` is `Ord` precisely so `cfg.stop_after <= Phase::Login` works directly.
- The summary uses `tracing::info!` so it appears at default log levels. For quiet crash-hunt runs, `RUST_LOG=bot=warn` hides it.

Commit: `feat(bot): coordinator with bounded duration + tracing summary`.

## Task C7 — Iterative live-server verification (DO THIS AFTER EVERY TASK)

This is not a single step — it is a **verification discipline** the LLM must follow after completing each of C2 through C6. After each task:

1. Build: `cargo build -p bot 2>&1` — fix any compile errors.
2. Run the appropriate phase-limited command against the live server. Each command targets exactly the milestone the just-completed task adds:

| After task | Run command | Expected `overall:` |
|---|---|---|
| C2 (config + HTTP API) | `cargo run -p bot -- --bots 1 --stop-after login --duration-secs 15` | `PASS` — `registered=true logged_in=true` |
| C3 (WebTransport) | `cargo run -p bot -- --bots 1 --stop-after stream --duration-secs 15` | `PASS` — `stream_ok=true` |
| C4 (input gen) | No live run needed — unit-testable locally, no server interaction. |
| C5 (bot state machine) | `cargo run -p bot -- --bots 2 --stop-after play --duration-secs 20` | `PASS` — both bots: `inputs_sent>0 exit=duration_reached` |
| C6 (coordinator) | `cargo run -p bot -- --bots 3 --stop-after play --duration-secs 20` | `PASS` — 3 bots: all milestones true |

3. Look for the `"swarm complete"` info line at the end of the output. It contains `overall=PASS` or `overall=FAIL`. If `FAIL`, read the per-bot `"bot result"` lines above it to identify which milestone failed, then fix the issue in the relevant module. Re-run.
4. Only move to the next task once the verification command shows `PASS`.

**Critical notes:**
- `GameHandle::min_players` is 2 (see `backend/src/game/ffi.rs:500`), so `--bots 1` will never trigger a game loop — it only exercises auth/lobby REST. Use `--bots 2+` for any test that includes `--stop-after stream` or `--stop-after play`.
- If the live server returns unexpected JSON shapes (wrong field names, different response structure), the error will appear in both the tracing logs and the summary `exit_reason` field. Fix `api.rs` field names to match what the server actually sends, not what the plan assumed.
- If the WebTransport handshake fails in C3, the first thing to check is how the frontend client opens the connection — read the frontend TS source for the exact URL path, headers, and query params.
- Keep runs short (≤30s) and bots few (≤4). The live server is shared infrastructure.

## Task C8 — Full integration test: 3 bots, 30 seconds

After all tasks pass their individual phase verifications:

cargo run -p bot -- --bots 3 --stop-after play --duration-secs 30
```

## Task C9 — Crash hunt (human-driven, not LLM-driven)

Once Tracks A + B are deployed to the target server and the bot is verified, a **human** runs unbounded sessions to reproduce the crash:

```bash
# 4 bots, deathmatch, no timeout — runs until killed or a crash is detected
cargo run -p bot -- --base-url https://transcendence.kajus.cloud --bots 4 --mode deathmatch --duration-secs 0

# Or against a local dev build with Track A/B logging active

```

Watch the **server's** logs (not the bot's). With Tracks A and B live, any crash produces:
1. `!!! game-core crash caught: SIGSEGV last_log=<file:function>` on stderr.
2. `game-core crashed — tearing down lobby` from Rust tracing.
3. The server stays alive for the next run.

Once repro is confirmed, dial down to `--bots 3` and loop runs to tighten the repro window. The `last_log=` field in the crash banner is the primary lead — it names the exact C++ function the engine was in at the moment of death.
